## 🎯 About The Project

[**Taqaddum (تقدّم)**](https://task-tracker-taqaddum.vercel.app/) means "**Progress**" in Arabic. This project was born from a deeply personal need: to create a unified platform for tracking daily Islamic practices, academic goals, and personal development. 

### The Story Behind Taqaddum

> *"This was not meant to be a 'Just for show' type of project. I don't even like Javascript. But our likings mean nothing when our deed or work is faulty. This is an acknowledgement that we are not good humans. And it's not okay to be okay with this version. We gotta change. Be the best possible version of ourselves. And it is not even a good deed. It is a basic responsibility."*
>
> — Project Author

In one word: **Being a better Muslim is the promise we made to our Creator.**

This application helps users:
- ✅ Track 5 daily prayers (Salah) - even if Qaza (missed)
- ✅ Monitor academic & coding progress

- ✅ See separate set of reports based on your custom task completion
- ✅ Stay accountable with fitness goals
- ✅ Learn core Islamic principles daily
- ✅ Fight addictions & maintain a clean lifestyle
- ✅ Collaborate with team members and stay motivated together
- ✅ Visualize progress through comprehensive reports and charts

**Note**: The following instructions are provided for transparency and learning purposes. Running your own instance requires explicit permission from the author.

1. **Clone the repository**
   ```bash
   git clone https://github.com/mehedyk/task-tracker-taqaddum.git
   cd task-tracker-taqaddum
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   ```bash
   # Create .env.local file
   touch .env.local
   ```

4. **Add your Supabase credentials to `.env.local`**
   ```env
   REACT_APP_SUPABASE_URL=your-supabase-project-url
   REACT_APP_SUPABASE_ANON_KEY=your-supabase-anon-key
   ```

5. **Set up the database** (see [Database Setup](#database-setup) below)

6. **Start the development server**
   ```bash
   npm start
   ```

7. **Open your browser**
   ```
   http://localhost:3000
   ```

## 🔌 API Documentation

### Supabase Tables Schema

#### **profiles**
```typescript
{
  id: UUID (Primary Key, Foreign Key to auth.users),
  name: TEXT,
  created_at: TIMESTAMPTZ (auto-generated),
  updated_at: TIMESTAMPTZ (auto-updated)
}
```

**Purpose**: Stores user profile information  
**RLS**: Users can only view/update their own profile  
**Auto-created**: When user signs up via trigger

---

#### **tasks**
```typescript
{
  id: UUID (Primary Key),
  user_id: UUID (Foreign Key to auth.users),
  task_id: TEXT,                          // e.g., 'salah', 'coding', 'fitness'
  task_name: TEXT,                        // Display name
  task_type: TEXT,                        // 'simple' | 'expandable' | 'subtask'
  parent_id: TEXT | NULL,                 // For subtasks only
  date: DATE,                             // Which day this task is for
  completed: BOOLEAN (default: false),
  icon: TEXT | NULL,                      // Emoji icon
  is_parent: BOOLEAN (default: false),    // True for parent tasks
  link: TEXT | NULL,                      // GitHub link for coding tasks
  created_at: TIMESTAMPTZ,
  updated_at: TIMESTAMPTZ
}
```

**Purpose**: Stores main daily tasks (Salah, Academic, Fitness, etc.)  
**Indexes**: user_id, date, task_id, parent_id  
**RLS**: Users can CRUD their own tasks, view all tasks for team reports

---

#### **custom_task_templates**
```typescript
{
  id: UUID (Primary Key),
  user_id: UUID (Foreign Key to auth.users),
  task_name: TEXT,
  icon: TEXT (default: '📝'),
  created_at: TIMESTAMPTZ,
  updated_at: TIMESTAMPTZ
}
```

**Purpose**: Stores user's recurring custom task templates  
**RLS**: Users can only access their own templates  
**Usage**: Templates auto-create daily instances in `custom_tasks`

---

#### **custom_tasks**
```typescript
{
  id: UUID (Primary Key),
  user_id: UUID (Foreign Key to auth.users),
  task_name: TEXT,
  icon: TEXT (default: '📝'),
  date: DATE,
  completed: BOOLEAN (default: false),
  template_id: UUID | NULL (Foreign Key to custom_task_templates),
  created_at: TIMESTAMPTZ,
  updated_at: TIMESTAMPTZ
}
```

**Purpose**: Stores daily instances of custom tasks  
**RLS**: Users can only access their own custom tasks  
**Template Link**: If created from template, template_id is set

### Key API Operations

#### **Authentication**

```javascript
// Sign Up
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'securepassword',
  options: { 
    data: { 
      full_name: 'User Name' 
    } 
  }
});

// Sign In
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'securepassword'
});

// Sign Out
await supabase.auth.signOut();

// Get Current Session
const { data: { session } } = await supabase.auth.getSession();

// Listen to Auth Changes
supabase.auth.onAuthStateChange((event, session) => {
  if (event === 'SIGNED_IN') console.log('User signed in');
  if (event === 'SIGNED_OUT') console.log('User signed out');
});
```

---

#### **Task Operations**

```javascript
// Get today's tasks
const today = new Date().toISOString().split('T')[0];
const { data, error } = await supabase
  .from('tasks')
  .select('*')
  .eq('user_id', userId)
  .eq('date', today)
  .order('created_at', { ascending: true });

// Create daily tasks from templates
const dailyTasks = [
  {
    user_id: userId,
    task_id: 'salah',
    task_name: 'Salah (At Least Qaza)',
    task_type: 'expandable',
    date: today,
    completed: false,
    icon: '🕌',
    is_parent: true
  },
  // ... more tasks
];

const { error } = await supabase
  .from('tasks')
  .insert(dailyTasks);

// Toggle task completion
const { error } = await supabase
  .from('tasks')
  .update({ 
    completed: true,

const weekData = [];
```

### Helper Functions

```javascript
// Calculate completion rate
function calculateCompletionRate(tasks, taskTemplates) {
  const mainTasks = tasks.filter(t => t.is_parent || t.task_type === 'simple');
  const completedMainTasks = mainTasks.filter(t => {
    if (t.task_type === 'simple') return t.completed;
    
    const subtasks = tasks.filter(st => st.parent_id === t.task_id);
    
    // Academic/IT: ANY ONE subtask
    if (t.task_id === 'coding') {
      return subtasks.length > 0 && subtasks.some(st => st.completed);
    }
    
    // Others: ALL subtasks
    return subtasks.length > 0 && subtasks.every(st => st.completed);
  });
  
  const total = Math.max(mainTasks.length, taskTemplates.length);
  const completed = completedMainTasks.length;
  return total > 0 ? Math.round((completed / total) * 100) : 0;
}

// Calculate streak
function calculateStreak(weeklyData) {
  let currentStreak = 0;
  let bestStreak = 0;
  let tempStreak = 0;
  
  for (let i = weeklyData.length - 1; i >= 0; i--) {
    if (weeklyData[i].percentage >= 70) {
      tempStreak++;
      if (i === weeklyData.length - 1) currentStreak = tempStreak;
    } else {
      bestStreak = Math.max(bestStreak, tempStreak);
      tempStreak = 0;
    }
  }
  bestStreak = Math.max(bestStreak, tempStreak);
  
  return { currentStreak, bestStreak };
}
```
